# Oni Design Principles

This document defines the core design principles that guide the creation of all web pages generated by Oni. The goal is to produce interfaces that look premium, feel state-of-the-art, and maintain an agency-level quality bar ($50,000+ value).

---

## 1. High Visual Impact & First Impression
*   **The Hero Wow Factor**: The top section (hero) of any website must hook the visitor instantly. Use large typography, subtle floating decorative elements (such as glassmorphism badges, floating background blur orbs), and strong contrasting layouts.
*   **No Placeholders**: Never use generic lorem-ipsum copy. The content must always feel real, describing actual menu items, rooms, or features with realistic pricing, names, and reviews.
*   **Generous Padding**: Give elements breathing room. Use deep section padding (`padding: 6rem 1.5rem` or `8rem 2rem`) to establish a luxury aesthetic.

## 2. Dynamic & Tactile Interactivity
*   **Micro-Animations**: Elevate static sections with hover states. Cards should translate slightly upward (`transform: translateY(-8px)`) and change border opacity or shadow depth on focus.
*   **Interactive Controls**: Build pure HTML/CSS interactive elements like tabs, slide-out sidebars, and toggle grids to show details without page refreshes.
*   **Smooth Motion**: Transitions must feel fluid and natural. Avoid harsh instant state changes; use cubic-bezier transitions (`transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1)`).

## 3. High-Fidelity Color Systems
*   **Bespoke Palettes**: Adopt curated palettes matching the brand type. Avoid generic colors (basic red, blue, green). Utilize deep rich backgrounds (e.g., `#0F0F1A` for dark mode Tech, `#1A0A0A` for Restaurants) with fine accent highlights.
*   **Luminous Depth**: Layer content using translucent backgrounds (glassmorphism) over glowing backdrop shapes (orbs) to create depth and modern dimension.

## 4. Typography Hierarchy & Scales
*   **Contrast Pairing**: Pair a highly expressive display typeface (e.g., *Playfair Display* or *Cormorant Garamond*) for headlines with a clean, readable sans-serif (e.g., *Inter* or *Jost*) for body content.
*   **Fluid Typography**: Use CSS `clamp()` for responsive headings to avoid layout breakage on mobile devices while maximizing screen real estate on desktop.

## 5. Mobile-First Layouts
*   **Fluid Grids**: Leverage CSS Grid and Flexbox for native responsiveness. Avoid rigid pixel-width grids.
*   **Responsive Modals & Navigation**: Ensure navigation hamburger toggles, cards, and modal popups scale gracefully to small screens with zero overlapping elements.
